Core Concept

ATA 34 is a navigation data engine→It provides position, attitude, direction, and hazard awareness.big idea
Navigation is not just radios→It is sensors, computers, receivers, display logic, and safety systems working together.
Bad nav often means bad data→A display warning may be the end of the chain, not the root problem.sensor first
🧩

System Breakdown

Four big categories→Air data/inertial, landing systems, radio navigation, and awareness systems.
System of systems→ADIRS, FMGC, MMR, receivers, TCAS, EGPWS, and WXR all exchange data.integration
ATA 34 feeds critical systems→Autoflight, displays, warnings, landing guidance, terrain, and traffic awareness rely on it.
📡

ADIRS — Core System

ADIRS is the heart of ATA 34→It uses ADIRUs plus pitot/static, AOA, TAT, and ADM data.most important
Three independent ADIRUs→They provide redundancy and cross-checking between sources.
Two parts inside each ADIRU→ADR handles air data. IR handles attitude, heading, and position.
🌬️

ADR vs IR

ADR = Air Data Reference→Airspeed, altitude, temperature, and pressure-based data.
IR = Inertial Reference→Pitch, roll, heading, attitude, and inertial position.
ADR and IR can fail independently→Do not treat ADIRS as one single failure bucket.exam + line
🔁

ADIRS Redundancy & Switching

Three systems cross-check each other→Disagreement can cause CHECK ATT or CHECK HDG messages.
Normal source logic→Captain side normally uses ADIRU 1, F/O side normally uses ADIRU 2.
ADIRU 3 is backup→ATT HDG and AIR DATA switching can select alternate data sources.source select
🔧

ADIRS Sensor Flow

Pitot/static data becomes digital→Pitot/static pressure goes to the ADM, which converts pressure into digital data.ADM role
Data path→Pitot/static → ADM → ADIRU → ARINC data buses → DMC/FWC/FMGC.
Most faults start upstream→Probes and ADMs fail or get blocked more often than the ADIRU core.
⏱️

ADIRS Alignment

Must align on ground and stationary→Movement or wrong position input can create navigation errors later.
Alignment takes time→The system needs time to establish accurate inertial reference.
Bad alignment creates delayed problems→The aircraft may appear okay at first, then show nav drift or disagreement later.watch setup
🛬

MMR / ILS / GPS

MMR combines ILS and GPS→The Multi-Mode Receiver provides landing guidance and GPS position data.
ILS gives precision landing guidance→Localizer is lateral guidance. Glide slope is vertical guidance.precision
ILS can override other guidance→During approach, valid ILS guidance becomes a high-priority reference.
GPS supports FMS position→GPS provides position, ground speed, and track data for navigation blending.
🛰️

GPS / FMS Position Logic

FMS position is hybrid→The aircraft can blend GPS, IR, and radio navigation into a computed position.not GPS only
GPS PRIMARY LOST does not mean lost navigation→The aircraft can revert to IRS and radio navigation data.
Disagreement messages matter→FMS/GPS disagreement means the data sources do not agree enough to trust blindly.
📻

VOR / DME / ADF

VOR is angular navigation→It provides bearing and course deviation based on phase difference.
DME measures slant range→Distance is calculated from signal time delay.time of flight
ADF points to NDB→ADF is lower priority and less accurate, but still tested and useful to understand.
📏

Radio Altimeter

RA is height above terrain→Radio altitude is not the same as barometric altitude.critical distinction
Used for landing and GPWS→RA data is critical for autoland, callouts, terrain warning, and low-altitude logic.
Low-altitude accuracy matters→Bad RA data can affect multiple landing and warning systems.
🌩️

Weather Radar & PWS

Weather radar detects weather hazards→It detects precipitation/weather returns and can support turbulence and terrain-related displays depending on configuration.
Radar needs ADIRS attitude→Attitude data stabilizes the radar beam so the display makes sense.ADIRS link
Predictive windshear is high priority→PWS warns of windshear ahead during low-altitude, high-risk phases.
✈️

TCAS

Traffic collision avoidance→TCAS uses transponder data to detect and avoid nearby aircraft.
TA and RA→Traffic Advisory warns of traffic. Resolution Advisory commands vertical avoidance action.safety alert
Coordinates with other aircraft→TCAS logic coordinates advisories so aircraft do not maneuver into each other.
⛰️

EGPWS

Terrain warning system→EGPWS uses GPS, RA, ADIRS, and terrain logic to warn of terrain threats.terrain safety
Multiple warning modes→Sink rate, terrain closure, altitude loss, config errors, and below glide slope logic.
Predictive safety system→EGPWS is not just reactive; it predicts terrain risk using navigation data.
🔧

Maintenance Reality

Most common faults start at sensors→Pitot/static blockage, ADM faults, AOA sensor issues, GPS loss, antennas, and receiver faults are common.
ADIRU core failure is less common→Do not shotgun an ADIRU before checking sensors, ADMs, wiring, and data disagreement.don’t shotgun
Cross-check left vs right→Compare captain/F/O/standby data to locate the bad source.

🔧 Pro Troubleshooting Flow

1. ECAMCheck NAV faults and related messages.
2. IdentifyADR, IR, radio nav, GPS, RA, radar, TCAS, or EGPWS.
3. Cross-CheckCompare left, right, and standby data.
4. CFDSUse fault history before replacing parts.
5. Trace SourceSensor → ADM/receiver → ADIRU/MMR → FMGC → display.
