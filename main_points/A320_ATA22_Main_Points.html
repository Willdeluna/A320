Core Concept — What ATA 22 Does

ATA 22 = automatic flight control + guidance→It controls lateral path, vertical path, and thrust automation.
AFS flies using flight plan + sensor inputs→The system is only as good as the data feeding it.data matters
AFS does not move surfaces directly→It sends commands to EFCS and FADEC.
🔄

System Architecture

Basic flow: sensors → FMGC → EFCS/FADEC→Then aircraft response feeds back into the system.
Main components→FMGC, FAC, FCU, MCDU, autopilot, flight director, and autothrust.
AFS is a system-of-systems→Problems often involve data, modes, or connected systems.big picture
🖥️

FMGC — Main Brain

FMGC = Flight Management + Flight Guidance→FM plans the flight. FG flies the guidance.FM vs FG
FM handles flight plan, navigation, and performance→This is the long-term planning side.
FG handles AP, FD, and A/THR→This is the active flying/guidance side.
🧬

Redundancy + Fail Modes

Two FMGCs and two FACs→Redundancy lets the system keep operating after certain failures.
Fail operational→System can continue the operation after one failure, like CAT 3 DUAL logic.autoland
Fail passive→System disconnects safely after a failure, like CAT 3 SINGLE capability.
⚠️

FMGC Modes

Normal mode = both FMGCs synchronized→This is the expected healthy mode.
Independent mode = computers disagree→Mismatch does not always mean total failure.don’t jump
Single mode = one FMGC failed→Capability remains, but redundancy is reduced.
🛡️

FAC — Protection + Yaw

FAC handles yaw damper and rudder trim→It helps stabilize yaw and coordinate aircraft response.
Rudder travel limit changes with speed→High speed means less rudder deflection allowed.
FAC computes protections→Speed limits, alpha floor, windshear, and envelope-related functions.protections
🎛️

FCU — Pilot Short-Term Control

FCU is on the glareshield→Pilots use it for speed, heading, altitude, and mode control.
Pull = selected mode→Pilot overrides the managed flight plan value.pull = pilot
Push = managed mode→FMGC follows the flight plan/performance logic.push = plan
📋

MCDU — Long-Term Planning

MCDU edits the flight plan→Routes, performance, navigation data, and constraints live here.
MCDU is not the same as FCU→MCDU plans. FCU commands short-term tactical changes.
✈️

Autopilot

AP controls pitch and roll→Yaw-related stabilization is handled through FAC functions.
Engages after about 5 seconds airborne→System needs valid flight conditions before engagement.
Always one lateral + one vertical mode→Read both modes to understand what the airplane is doing.mode awareness
Pilot override disconnects AP→Sidestick, rudder input, or faults can disconnect the system.
🛬

Dual Autopilot + Autoland

Dual AP is required for CAT 3 DUAL→Two autopilots provide fail-operational capability.
One AP = reduced autoland capability→Usually fail-passive, not fail-operational.
Approach capability shows on FMA→Do not guess CAT capability — read the FMA.FMA first
📐

Flight Director

FD shows guidance bars on the PFD→It shows what the autopilot would command.
Works with or without autopilot→Pilot can manually fly the FD guidance.
Symbols depend on mode→Bars vs flight path vector can change depending on selected guidance.
⚡

Autothrust

A/THR controls thrust automatically→FMGC sends thrust commands to FADEC.
A/THR does not physically move the levers→This trips people up if they expect Boeing-style movement.Airbus logic
Engaged does not always mean active→ENGAGED + ACTIVE = auto thrust. ENGAGED + NOT ACTIVE = manual thrust.critical
Modes include SPEED/MACH, THR, and RETARD→Know whether the system is controlling speed or commanding thrust.
🚨

Alpha Floor Protection

Triggered by high angle of attack→The system protects the aircraft from getting too slow/high AOA.
Commands TOGA thrust automatically→This is a protection event, not normal speed mode behavior.safety logic
📊

FMA — The #1 Troubleshooting Display

FMA lives on the PFD→It tells you what the system is doing, armed to do, or no longer able to do.
Five columns→A/THR, vertical, lateral, approach capability, and engagement status.read left to right
Color logic matters→Green active, blue armed, white advisory, amber/red fault.
White box means recent change→Great clue for mode reversions, pilot selections, or system changes.
🔧

Maintenance Reality

Most faults are data mismatch or mode confusion→ADIRS mismatch, FMGC disagreement, FAC faults, and FCU issues are common.
Full system failure is rare→Think inputs, synchronization, and logic before parts-cannon mode.no parts cannon
FAC loss usually means loss of protections→Not automatic loss of basic aircraft control.
🧭

Troubleshooting Flow

Step 1: Check FMA→Mode and status tell the story first.
Step 2: Identify AP, FD, or A/THR issue→Narrow the problem to the affected function.
Step 3: Check FMGC/FAC status→Look for sync, independent, single, or fault conditions.
Step 4: Check inputs→ADIRS and sensor data drive the logic.mode → computer → input
