Core Concept

ATA 23 = all aircraft communication→Voice, audio, data, cabin calls, PA, interphone, and airline messages.
Communication is a network→Crew to ATC, crew to crew, crew to passengers, and aircraft to airline.big picture
Troubleshoot by path→Source → routing → output. Don’t parts-cannon a radio before checking routing.
🔄

System Architecture

Basic flow→Microphone → AMU → radio / interphone / PA → headset, speaker, or antenna.
Multiple communication paths→Radio, interphone, PA, CIDS, ACARS, and ATSU all share logic.
Most faults are routing faults→Selection, audio path, AMU, or bus data can look like bad hardware.MX clue
📻

VHF System

VHF = primary daily communication→Used for ATC, company, ground, and normal line operations.
VHF 1, VHF 2, VHF 3→VHF 3 is often used for data link like ACARS / ATSU.
RMP tunes, ACP routes audio→Frequency problem? Think RMP. Hearing/transmit problem? Think ACP/AMU.don’t mix them up
🌊

HF System

HF = oceanic / long range→Used where VHF line-of-sight coverage is not enough.
Transceiver + coupler + antenna→The coupler tunes the antenna system for the selected frequency.
HF is noisy by nature→Interference, tuning faults, and antenna issues are common suspects.known headache
🎛️

RMP — Radio Management Panel

RMP = tuning head→It controls frequency selection; it is not the actual radio.
Controls VHF, HF, and backup nav tuning→Any RMP can control radios through reconfiguration logic.
Blank RMP display→Turn it off for reconfig; system can route control through remaining panels.line trick
🎚️

ACP — Audio Control Panel

ACP = audio routing panel→It selects transmit channel, receive channels, volume, PA, and interphone.
Only one transmit channel at a time→You can listen to several channels, but transmit on one selected source.high value
Pushbutton vs knob logic→Pushbuttons select transmit. Knobs select and adjust reception.
🧩

AMU — Audio Management Unit

AMU = traffic controller for audio→Routes microphone inputs, radio outputs, interphone audio, and SELCAL.
Mic to radio / PA / interphone→Same input can go different places depending on ACP and system logic.
Radio to headset / speaker→If audio is missing, trace the AMU path before blaming the radio.routing hub
☎️

Interphone + Ground Call

Three interphone types→Flight interphone, service interphone, and cabin interphone.
Service interphone = ground ops→Connects cockpit/cabin with maintenance stations around the aircraft.
Ground call is loud→Horn and light alert crew. Stand clear when testing near nose gear.safety
📢

Passenger Address — PA

PA sends announcements to cabin speakers→Inputs can come from cockpit, cabin crew, or recorded messages.
Priority matters→Cockpit overrides cabin crew, recorded messages, and music.priority logic
Volume adjusts automatically→Engine running and decompression logic increase PA volume.
🏠

CIDS — Cabin System

CIDS = cabin communication and control→Controls PA, interphone, lighting, passenger signs, and calls.
Directors provide redundancy→DIR 1 active, DIR 2 standby; auto switch if needed.
DEUs split cabin functions→Type A handles passenger devices; Type B handles crew/sensor interfaces.partial failures
🛰️

ACARS / ATSU Data Link

ACARS = airline data link→Sends OOOI times, maintenance data, and messages, often through VHF 3.
ATSU = advanced communication router→Handles CPDLC, ADS, and ATC data link functions on newer aircraft.
Interfaces with many aircraft systems→FMGC, DMU, FWC, and SDAC can all feed data.system-of-systems
🔔

SELCAL + Static Dischargers

SELCAL alerts crew to calls→It compares incoming code to the aircraft code and triggers light/buzzer.
Crew does not need constant listening→Especially useful on HF and long-range communication.
Static dischargers improve radio clarity→After lightning strikes, inspect them early.MX first look
🎙️

CVR — Cockpit Voice Recorder

Records cockpit audio→Crew comms, interphone, PA, and ambient cockpit sound.
Stores the last 2 hours→Used for incident and accident investigation.
🔧

Troubleshooting Flow

Step 1: identify the system→Radio, PA, interphone, data link, or cabin communication?
Step 2: check RMP and ACP→Frequency and routing issues are the classics.
Step 3: trace input to AMU to output→Then check power, antennas, coax, and bus data.source → routing → output
